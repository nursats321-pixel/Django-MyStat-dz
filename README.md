## Часть 2: DevTools 
Вам нужно запустить проект локально, войти в систему и провести инспекцию своего бэкенда через DevTools

Скриншот №1 (Проверка метода и статуса):
Откройте вкладку Network (Сеть). Отправьте любое новое сообщение в чат. Поймайте момент отправки и сделайте скриншот заголовков (Headers) этого запроса. На скриншоте должно быть четко видно две строчки из блока General:

Request Method: POST

Status Code: 302 Found (или редирект)

Скриншот №2 (Взлом токена):

Зайдите во вкладку Application (Хранилище) Cookies и скопируйте ваш access_token

Зайдите на сайт jwt.io и вставьте его туда

Измените в правом окне (Payload) значение "is_staff": false на true

Скопируйте получившийся поддельный токен, вернитесь в браузер и вручную замените им ваш настоящий токен в Cookies.

Обновите страницу чата и сделайте скриншот того, что вам ответил сервер (там должен быть белый экран и статус 403, который мы прописали во views.py)
